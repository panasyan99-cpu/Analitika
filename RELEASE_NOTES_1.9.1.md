# Analitika Web 1.9.1 — Cloud Order Library

Версия добавляет полноценное облачное меню заказов поставщику и безопасное полное удаление сохранённого заказа.

## Незавершённые заказы

В модуле **«Заказ поставщику»** появилась заметная кнопка **«Незавершённые заказы»**. Она доступна на стартовой странице и в боковом меню.

Список загружается напрямую из Cloudflare R2 и показывает для каждого заказа:

- исходное название Excel;
- дату создания;
- дату последнего изменения;
- отдельные режимы «Камни» и «Жемчуг»;
- текущее количество выбранных изделий;
- количество SKU;
- количество позиций Limited Order;
- текущий этап;
- статус `draft` или `completed`.

Кнопка **«Продолжить заказ»** восстанавливает исходный Excel и соответствующий облачный черновик без ручного импорта JSON.

## Компактный облачный индекс

Добавлен объект `orders-index.json`. Обычное открытие списка использует один компактный JSON и не скачивает все исходные Excel или manifest-файлы.

Кнопка **«Обновить список»** выполняет принудительное перестроение индекса по manifest-файлам. Это также автоматически переносит старые сохранения в новый формат индекса.

## Завершённые заказы

Заказ можно отметить завершённым в блоке итогового Excel. Завершённые заказы скрываются из облачного меню по умолчанию и отображаются после включения переключателя **«Показать завершённые»**. Завершённый заказ можно вернуть в черновики.

## Полное удаление

У каждой карточки есть кнопка **«Удалить»** с обязательным вторым подтверждением.

После подтверждения удаляются:

- исходный Excel;
- `manifest.json`;
- черновики «Камни» и «Жемчуг»;
- Limited Order и размеры колец внутри черновиков;
- любые резервные версии и дополнительные объекты внутри workspace;
- запись из `orders-index.json`;
- локальные записи SQLite;
- локальные копии Excel;
- session-state и кэш разбора файла.

Облачная запись удаляется из индекса только после повторной проверки, что под префиксом workspace не осталось объектов. При частичном удалении приложение показывает оставшиеся ключи и сохраняет заказ в списке.

## Интерфейс карточки изделия

Сохранены согласованные изменения интерфейса:

- «Общий остаток» показывается всегда, включая ноль;
- TT и 63 при нуле выводятся компактно;
- отрицательные остатки подсвечиваются как ошибка данных;
- рекомендация показывается крупной цифрой;
- Limited Order расположен под остатками и не растягивает карточку;
- служебный NTR2 не выводится в карточке.

## Совместимость

Старые manifest-файлы поддерживаются. При первом обновлении списка индекс строится автоматически. Ручное поле восстановления JSON удалено из интерфейса; аварийный ZIP-резерв сохранён.
