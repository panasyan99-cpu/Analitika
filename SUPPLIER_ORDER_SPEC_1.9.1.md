# Спецификация «Заказ поставщику» 1.9.1

## 1. Модель хранения

Каждый исходный Excel получает `source_hash` и отдельный префикс:

```text
<prefix>/workspaces/<source_hash>/
```

Внутри могут находиться любые связанные объекты, включая:

```text
source.xlsx
manifest.json
draft-stones.json
draft-pearls.json
history/...
backup-...
```

Компактный каталог расположен отдельно:

```text
<prefix>/orders-index.json
```

## 2. Состав индекса

Для каждого `source_hash` индекс хранит только метаданные, необходимые для списка:

- `source_name`;
- `workbook_key` и размер;
- `created_at`;
- `updated_at`;
- общий статус;
- метаданные режимов «Камни» и «Жемчуг»:
  - дата создания и изменения;
  - количество SKU;
  - общее количество изделий;
  - количество Limited Order;
  - этап;
  - статус.

Исходный Excel и полный черновик при построении списка не скачиваются.

## 3. Восстановление

При нажатии **«Продолжить заказ»**:

1. приложение читает manifest выбранного `source_hash`;
2. при отсутствии локальной копии скачивает исходный Excel из R2;
3. парсит Excel с исходным хешем;
4. загружает `draft-stones.json` или `draft-pearls.json`;
5. восстанавливает количества, ручной режим, Limited Order, размеры, сверку остатков, выбранный камень, этап и статус;
6. открывает рабочий интерфейс выбранного режима.

## 4. Статусы

Поддерживаются статусы:

- `draft` — незавершённый заказ;
- `completed` — завершённый заказ.

Общий workspace считается завершённым, только когда все существующие режимы завершены. Завершённые workspace скрываются по умолчанию.

## 5. Полное удаление

Удаление выполняется по всему префиксу workspace, а не по фиксированному списку файлов.

Алгоритм:

1. получить все ключи с префиксом workspace;
2. удалить их пакетами до 1000 объектов;
3. повторно получить список ключей;
4. при наличии остаточных объектов остановить операцию и показать их пользователю;
5. только после пустой проверки удалить запись из индекса;
6. удалить локальные черновики и локальные Excel;
7. очистить session-state и кэши.

Это гарантирует удаление будущих резервных форматов без изменения алгоритма.

## 6. Интерфейс

Карточка облачного заказа содержит:

- название файла;
- статус и место хранения;
- даты создания и изменения;
- общие KPI: изделия, SKU, Limited Order;
- отдельную строку каждого режима;
- кнопку «Продолжить заказ»;
- кнопку «Удалить»;
- двухэтапное подтверждение полного удаления.

Ручной uploader JSON отсутствует.
