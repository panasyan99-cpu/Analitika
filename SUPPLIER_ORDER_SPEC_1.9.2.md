# Спецификация «Заказ поставщику» 1.9.2

## 1. Остатки изделия

Для каждой позиции хранятся два разных показателя.

### 1.1. Общий остаток для интерфейса

```text
Общий остаток = Всего − магазин 20
```

Правила:

- кубик показывается всегда, включая ноль;
- TT/Outlet входит в общий остаток;
- 63 входит в общий остаток;
- TT и 63 показываются отдельными кубиками как детализация;
- TT и 63 не вычитаются повторно;
- магазин 20 в карточке не показывается.

### 1.2. Остаток для рекомендаций

```text
working_stock = Всего − магазин 20 − магазин 63 − Princess Hang
```

Этот показатель используется только внутренними правилами заказа и не может отображаться под подписью «Общий остаток».

## 2. Нормализация магазинов

Магазин 20 распознаётся по заголовкам:

- `20`;
- `20NDC`.

Магазин 63 распознаётся по заголовкам:

- `63`;
- `63NDC`;
- `63.1`;
- `63.2`.

Если в отчёте несколько колонок магазина 63, они суммируются.

TT определяется по Outlet/TT, но складская колонка `Stock TT` не считается магазином TT.

## 3. Выгрузки

В Limited Order Excel колонка «Всего остаток» содержит отображаемый общий остаток `Всего − магазин 20`.

Внутренний `working_stock` в эту колонку не выгружается.

## 4. Рекомендации

Все существующие правила рекомендаций продолжают использовать `working_stock`:

- двухмесячная потребность;
- заполнение TT;
- баланс комплектов;
- правило пусетов;
- проверка остатка перед заказом.

Изменение 1.9.2 не повышает и не понижает рекомендации автоматически — оно исправляет пользовательское отображение остатка и Limited Order Excel.
