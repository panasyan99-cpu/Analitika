# Аналитика 1.1.6 — Installer & Dual Update Channels

## Что готово

- Один Windows-установщик без необходимости устанавливать Python на компьютеры пользователей.
- Постоянный `AppId`: будущие версии обновляют существующую установку, даже если пользователь выбрал другую папку при первой установке.
- Пользовательские отчеты и настройки хранятся в `Документы/Analitika` и не удаляются при обновлении.
- Два канала обновлений:
  - **Stable** — только проверенные релизы.
  - **RC** — тестовые prerelease-версии и стабильные релизы.
- Приложение проверяет GitHub Releases, скачивает новый Setup и запускает его поверх текущей версии.
- GitHub Actions собирает `.exe`, установщик и SHA-256 checksum.

## Как выпустить тестовую версию

1. В `version.json` указать, например, `1.1.7-rc1`.
2. Commit + Push.
3. Создать и отправить тег:
   `v1.1.7-rc1`
4. GitHub Actions создаст prerelease. Его увидят только пользователи канала RC.

## Как выпустить стабильную версию

1. В `version.json` указать, например, `1.1.7`.
2. Commit + Push.
3. Создать и отправить тег:
   `v1.1.7`
4. GitHub Actions создаст обычный Release. Его увидят Stable и RC.

## Ручная сборка

Запустить `build_installer.bat` на Windows с установленными Python и Inno Setup.
Для пользователей Python не нужен — им передается только `Analitika_Setup_<version>.exe`.
