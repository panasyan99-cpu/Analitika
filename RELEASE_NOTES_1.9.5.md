# Analitika Web 1.9.5 — Stone Tiers and Minimum Batch

## Единая матрица рекомендаций

Рекомендация по каждому SKU сначала рассчитывается по продажам, TT, общему и расчётному остатку, ТВП и балансу настоящего комплекта. После этого применяется поправка по группе камня и правило минимальной автоматической партии.

Полную шкалу жемчуга без уменьшения используют:

- Ruby;
- все Sapphire;
- все Topaz;
- Moissanite;
- Emerald;
- Created Emerald;
- Onyx;
- Black Spinel;
- Green Agate.

Для остальных цветных камней базовая рекомендация уменьшается на одну единицу. Например, 5 становится 4, 4 становится 3, 6 становится 5.

## Минимальная автоматическая партия

После всех остальных расчётов действует единое правило:

- 0 → рекомендация отсутствует;
- 1 → рекомендация отсутствует;
- 2 → рекомендация 3;
- 3 и больше → количество не меняется.

Ручной ввод не ограничивается этим правилом.

## «Без комплекта»

Отсутствие товара в TT проверяется отдельно от общего сетевого остатка. Позиция с небольшим остатком больше не пропускается только потому, что достигнут базовый сетевой уровень.

При этом перекрёстная логика по ТВП по-прежнему отключена: отдельное кольцо в пути не создаёт рекомендацию для отдельных серёг и наоборот.

## Прозрачные причины

В карточке показывается причина корректировки количества:

- уменьшение на одну единицу для остальных цветных камней;
- подавление расчётной партии 1 шт.;
- повышение расчётной партии 2 шт. до минимальных 3 шт.

Собственный положительный ТВП продолжает полностью блокировать автоматический повторный заказ того же SKU.
