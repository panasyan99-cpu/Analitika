# Дополнение к ТЗ: постоянное хранение заказа

## Обязательное требование

Состояние заказа не должно зависеть от `st.session_state`, локального SQLite или файловой системы контейнера Streamlit.

## Источник истины

Приватное S3-совместимое хранилище:

- `workspaces/<source_hash>/source.xlsx` — неизменяемый исходный отчёт;
- `workspaces/<source_hash>/draft-stones.json` — заказ по камням;
- `workspaces/<source_hash>/draft-pearls.json` — заказ по жемчугу;
- `workspaces/<source_hash>/manifest.json` — имя файла, дата, этап, число выбранных позиций и общий объём.

## Идентификация

`source_hash` — SHA-256 содержимого Excel. Имя файла не влияет на идентификацию.

- одинаковые байты под другим именем открывают тот же заказ;
- изменённый файл с прежним именем создаёт новый чистый заказ;
- предыдущий заказ остаётся в истории и не загрязняет новый.

## Гарантии интерфейса

- зелёный статус подтверждает доступность облака;
- красный статус предупреждает, что безопасное хранение не настроено;
- при обязательном режиме загрузка нового отчёта блокируется без облака;
- каждое изменение сохраняется сразу;
- ручная кнопка выполняет повторную запись;
- после redeploy заказ доступен через «Продолжить сохранённый заказ»;
- восстановление загружает Excel из облака в новый временный runtime и затем поднимает черновик.
