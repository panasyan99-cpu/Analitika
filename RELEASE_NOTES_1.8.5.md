# Analitika Web 1.8.5

## Фактический NTR2, свободные имена файлов и конкретные Green Stones

Версия подготавливает модуль «Заказ поставщику» к новому формату входного отчёта.

### Основные изменения

- Если в исходном Excel присутствует колонка `NTR2`, приложение использует только фактическое значение из неё.
- Расчётный NTR2 сохранён исключительно как fallback для старых файлов без этой колонки.
- Несовпадение суммы магазинов с колонкой «Всего» показывается как предупреждение и не заменяет фактический NTR2.
- Имя загружаемого Excel может быть любым; формат определяется по структуре листа и колонок.
- Группа `Green Stones` используется только как крупный блок навигации.
- В карточках изделий, распределении размеров и итоговом Excel всегда сохраняется конкретный нормализованный камень: `Green Agate`, `Chrome Diopside`, `Created Emerald`, `Emerald` или `Peridot`.
- Добавлена нормализация `Green Agat` → `Green Agate`.
- Если исходный отчёт передаёт только `Green Stones`, конкретный камень восстанавливается по артикулу, когда в нём есть однозначное сокращение.
