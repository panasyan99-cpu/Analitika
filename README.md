# Analitika Web 1.2.5

Внутренняя аналитическая платформа Princess Jewelry для продаж, сравнения периодов, склада Baserow и формирования заказа Sonu.

## Режимы

- **Обычный отчет** — оперативная сводка, сеть, магазины, интерактивные срезы и поставщики.
- **Сравнение периодов** — сопоставление сети, магазинов, сегментов, камней, товарных групп и поставщиков в двух периодах.
- **Сувениры и касты на складе** — read-only аналитика Baserow по остаткам, фотографиям, проблемным позициям, движению и поставкам.
- **Заказ Sonu** — продажи по магазинам, средние продажи, браслеты, прогноз, матрица заказа и рекомендации.

Инструкции по подготовке входного файла размещаются внутри выбранного режима. Раздел **«О программе»** содержит только актуальное описание возможностей и автоматически обновляемую историю релизов.

## Общие правила

- денежные показатели сайта отображаются в USD;
- единый курс VND/USD задается на главном экране и применяется ко всем режимам;
- базовый курс — 26 300 VND за 1 USD;
- возвраты не учитываются в «Заказе Sonu»;
- обычная аналитика исключает возвраты, отмены и CHAIN;
- диаграммы защищены от случайного масштабирования и перемещения, подсказки сохранены;
- таблицы остаются сортируемыми;
- интерфейс адаптирован под ПК, iPad и смартфоны.

## Заказ Sonu

Навигация состоит из шести блоков:

1. **Продажи по магазинам**.
2. **Средние продажи** — скорость, нормализованная к 30 дням, и взвешенная средняя цена `Продажи / Количество`.
3. **Браслеты** — тип конструкции и отдельная аналитика камней внутри каждого типа.
4. **Прогноз продаж** — горизонт 30, 60 или 90 дней.
5. **Матрица заказа** — расчет точечного заказа под выбранный запас в месяцах.
6. **Рекомендации** — дефицитные и медленно оборачиваемые SKU, оценка общего покрытия.

Расчетный остаток Sonu равен **отгружено − продано**. Это оценка по входному отчету, а не подтвержденный фактический остаток склада.

## Склад Baserow

Складской режим читает таблицы сувениров, комплектующих, операций и поставок. Подключение задается в Streamlit Secrets:

```toml
[baserow]
url = "https://storage.princess-jewelry.com"
token = "READ_ONLY_TOKEN"
souvenirs_table_id = 642
components_table_id = 643
operations_table_id = 644
supplies_table_id = 645
```

Для токена достаточно права `Read`. Без секрета складской режим покажет инструкцию подключения, а остальные режимы продолжат работать.
