# Analitika Web 1.2.1

Стабильная внутренняя аналитическая платформа Princess Jewelry.

## Сценарий работы

1. Откройте отчет **«Продажи товаров»** в 1С.
2. Выберите период анализа.
3. Включите уровни: **Магазин**, **Номенклатурная группа**, **Камень / вставка**, **Поставщик**.
4. Сохраните выгрузку в Excel и загрузите ее в Analitika.

На стартовой странице доступны три раздела:

- **Обычный отчет** — формируется автоматически сразу после загрузки базовой выгрузки.
- **Сравнение периодов** — ожидает два одинаково настроенных отчета с разными периодами и запускается только после нажатия кнопки **«Запустить сравнительный анализ»**.
- **Сувениры и касты на складе** — получает read-only данные из Baserow и показывает изделия и компоненты, которые уже приняты, но ещё не переданы в бухгалтерию.

До формирования отчета в боковой навигации доступны только **«Загрузка»** и **«О платформе»**. После обработки появляются разделы выбранного режима.

## Разделы

1. **Оперативная сводка** — компактный режим для руководителя: сеть в одном экране, лидеры, структура сегментов, концентрации и крупнейшие поставщики.
2. **Сводка** — показатели сети, магазины, сегменты, диаграммы и выводы.
3. **Магазины** — детальная аналитика выбранного магазина.
4. **Интерактивная аналитика** — фильтры по магазину, сегменту, камню и номенклатурной группе.
5. **Поставщики** — сравнение поставщиков и детализация по магазинам, сегментам, камням и товарным группам.
6. **Сравнение периодов** — сеть, магазины, интерактивные срезы и поставщики в двух периодах.
7. **Сувениры и касты на складе** — остатки, фотографии, фильтры, ранние предупреждения, движение и поставки.
8. **О платформе** — инструкция, описание разделов и история обновлений.

Экспорт отключен. Поставщики с долей ниже 4,5% объединяются в Other только на круговых диаграммах; полная детализация сохраняется ниже. В разделе «О платформе» выводится история фактических обновлений вместо изменяемых планов.

Интерфейс адаптирован под ПК, iPad и смартфоны. На планшете KPI и фильтры перестраиваются в две колонки, парные диаграммы в портретном режиме показываются вертикально, а на телефоне основные блоки идут по одной колонке. Таблицы сохраняют сортировку и получают горизонтальную прокрутку.

Диаграммы работают в режиме просмотра: масштабирование, перетаскивание, выделение, изменение легенды и сохранение через панель Plotly отключены. Подсказки по наведению и касанию сохранены. Таблицы остаются интерактивными и поддерживают сортировку.


## Стабильность 1.1.7

- фильтры магазинов, интерактивной аналитики и поставщиков обновляют только свой раздел;
- Excel разбирается один раз и переиспользуется между повторными запусками и браузерными сессиями;
- кэш ограничен по времени и количеству отчетов;
- одновременно создается только выбранная детализация, а не содержимое всех скрытых вкладок;
- production должен быть развернут на **Python 3.12**.

## Warehouse analytics 1.2.0

Раздел **«Сувениры и касты на складе»** читает четыре таблицы Baserow в режиме only-read:

- Сувенирка — 642;
- Комплектующие — 643;
- Операции — 644;
- Поставки — 645.

Для Streamlit Cloud добавьте секреты:

```toml
[baserow]
url = "https://storage.princess-jewelry.com"
token = "READ_ONLY_TOKEN"
souvenirs_table_id = 642
components_table_id = 643
operations_table_id = 644
supplies_table_id = 645
```

Токен должен иметь только право `Read`. Он хранится в Streamlit Secrets и не попадает в GitHub или браузер руководителя. Данные кэшируются не более чем на 60 секунд; кнопка «Обновить данные» очищает кэш немедленно.
