# Analitika Web 1.4.0

Внутренняя аналитическая платформа Princess Jewelry для общего анализа продаж, сравнения периодов, склада Baserow и поставщика Sonu.

## Главное в версии 1.4.0

Раздел Sonu переведен на новый расширенный отчет с фактическим общим остатком сети. Значение остатка повторяется в строках разных магазинов для одного SKU, поэтому система берет его один раз на SKU и отдельно суммирует продажи по магазинам.

### Логика Sonu

- **Остаток сети** — одно значение на SKU, без суммирования дублей между магазинами.
- **Продажи за период** — сумма продаж SKU по всем магазинам.
- **Изделия, кроме браслетов** — группировка `группа камня → вид камня → категория изделия`.
- **Показатели** — количество SKU-моделей, остаток сети, продано за период, продажи в USD и потребность на 30/45/90 дней.
- **Браслеты с центральной композицией** — модели с затяжкой и модели на кольцах, где камни расположены преимущественно в середине и общий вес ниже.
- **Полный круг** — браслеты с камнями по всей окружности.
- **Неоднозначные браслеты** — распределяются 50/50; при нечетном количестве лишняя более продаваемая модель относится к центральной композиции.
- **Вид представления** — единый переключатель между карточками и таблицами.
- **Контроль данных** — несовпадающие повторные остатки одного SKU выводятся отдельным предупреждением и листом Excel.

### Полная выгрузка Sonu

Excel-файл включает:

1. Сводку.
2. Сеть по группам камней.
3. Камни и категории изделий.
4. Все SKU сети.
5. Типы браслетов.
6. Виды камней в браслетах.
7. Модели браслетов.
8. Продажи по магазинам.
9. Контроль расхождений остатка.
10. Исходные данные.

## Остальные режимы

- **Обычный отчет** — сеть, магазины, группы камней, товарные группы и поставщики.
- **Сравнение периодов** — сопоставление двух отчетов по основным аналитическим срезам.
- **Склад Baserow** — read-only аналитика сувениров, кастов, движения и поставок одной длинной страницей.

## Запуск

```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

Версия приложения хранится в `version.json`. История обновлений автоматически выводится из `CHANGELOG.md`.

## Глобальный фильтр металла

В сравнении периодов фильтр расположен сразу под блоком загруженных файлов. Переключатели «Серебро», «Золото и платина» и «Другое» одновременно пересчитывают все KPI, таблицы и диаграммы обоих периодов.
