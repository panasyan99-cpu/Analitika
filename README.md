# Analitika Web 1.6.0

Внутренняя аналитическая платформа Princess Jewelry для общего анализа продаж, сравнения периодов, склада Baserow и заказа Sonu.

## Единый формат обычного и сравнительного отчета

Основной входной отчет теперь строится в иерархии:

`Магазин → Камень / вставка → Проба → Номенклатурная группа`

Система автоматически определяет магазины и период, исключает CHAIN и не учитывает возвраты. Пустая проба сохраняется как «Не указано».

### Группы металла

- **Серебро** — B 925, 925, Ag, Silver и другие распознанные серебряные варианты.
- **Золото и платина** — любые AU / Gold и PT / Platinum.
- **Другое** — Other 0, пустая проба и все значения, не относящиеся к серебру, золоту или платине.

В сравнении периодов все три группы включены по умолчанию. Галочки являются глобальными: отключенная группа полностью исключается из KPI, количества, выручки, магазинов, таблиц, диаграмм и интерактивных срезов обоих периодов.

Отдельный блок **«Металлы и пробы»** содержит одну адаптивную диаграмму по штукам и USD и подробную таблицу до уровня камня и номенклатурной группы.

## Sonu

Sonu использует отдельный формат с общим остатком сети на SKU. Повторяющийся остаток по магазинам берется один раз, продажи суммируются по всей сети.

- общий управленческий отчет по продажам и остаткам;
- AI-приоритет заказа от «Очень нужно заказать» до «Не критично»;
- горизонт покрытия 30/45/90 дней;
- сводка `группа камней → вид камня → номенклатурная группа`;
- таблицы и карточки с настраиваемой сортировкой;
- браслеты «С затяжкой» и «Без затяжки (в круг)» с отдельными графиками и сводками;
- полная Excel-выгрузка и контроль повторяющихся остатков SKU.

## Склад Baserow

Read-only аналитика сувениров и комплектующих одной длинной страницей: обзор, сувениры, касты, позиции, требующие внимания, движение и поставки.

## Запуск

```bash
pip install -r requirements.txt
streamlit run streamlit_app.py
```

Версия хранится в `version.json`; раздел «О программе» автоматически формируется из каталога функций и `CHANGELOG.md`.
