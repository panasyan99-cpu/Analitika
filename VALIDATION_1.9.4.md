## Регрессии рекомендаций

Проверено синтетическими сценариями:

- слабый комплект: продажи 2, общий остаток 1, TT 0 → серьги 5, кольца 3;
- продажи 2, TT 1 или 2, общий остаток меньше 5 → рекомендация сохраняется;
- продажи 2, общий остаток 5 и изделие присутствует в TT → рекомендация 0;
- нулевой реальный комплект → 5 / 3 / 2 по типам;
- «Без комплекта» при 0/0 → 5 / 3 / 2 без упоминаний комплекта;
- «Без комплекта» при небольшом остатке и TT 0 → компактное пополнение 3 / 2 / 1–2;
- ТВП серёг 5, кольцо 1 → 3 кольца;
- ТВП серёг 10, кольца 0 → 6 колец;
- ТВП серёг 10, кольца 6 → 0;
- ТВП колец 4 → 5 серёг только внутри реального комплекта;
- в «Без комплекта» ТВП другого SKU не влияет на рекомендацию;
- собственный положительный ТВП всегда блокирует повторный автоматический заказ этого SKU.

## Интерфейс

Проверено, что:

- шапка больше не содержит фразу «рабочий остаток без 63 и 20»;
- рекомендация остаётся предложением и не попадает в заказ без подтверждения;
- для SKU с собственным ТВП доступна ручная кнопка дозаказа;
- общий остаток, TT и 63 отображаются по логике 1.9.3 без регрессии.
