# Analitika Web 1.9.3 — Unified Store 20 Logic

## Исправленная бизнес-логика

Магазины `20`, `20NDC` и `Princess Hang` являются одной и той же точкой. В версии 1.9.2 они могли учитываться как отдельные остатки, из-за чего внутренний расчёт мог вычитать один магазин дважды.

Версия 1.9.3 использует единый показатель:

```text
excluded_store_20 = 20 / 20NDC / Princess Hang
```

### Отображаемый общий остаток

```text
Общий остаток = Всего − excluded_store_20
```

TT/Outlet и 63 входят в общий остаток и дополнительно показываются отдельными кубиками.

### Остаток для рекомендации

```text
working_stock = Всего − excluded_store_20 − 63
```

Этот показатель используется только алгоритмом рекомендаций.

## Дедупликация синонимов

Если отчёт содержит более одной колонки из набора `20`, `20NDC`, `Princess Hang`, значения не суммируются. Используется максимальное неотрицательное значение, потому что все колонки обозначают одну точку.

Если ненулевые значения различаются, позиция получает предупреждение с перечислением конфликтующих колонок.

## Проверка на рабочем файле

Для SKU `KE15E518B-BS` из файла `Заказ.xlsx` подтверждено:

```text
Всего = 4
20 / Princess Hang = 0
Общий остаток = 4
TT = 1
63 = 1
Остаток для рекомендации = 3
```

## Ошибка NameError после деплоя

Код исправления старого `NameError` уже входит в репозиторий. Если ошибка остаётся на Streamlit Cloud, приложение использует смешанный или закэшированный набор файлов. Порядок полного обновления приведён в `DEPLOY_FIX_1.9.3.md`.
