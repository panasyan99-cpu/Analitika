# Analitika Web 1.8.8 — Durable Supplier Order Storage

## Главное исправление

Заказ поставщику больше не зависит от временного диска Streamlit. Исходный Excel и черновики сохраняются в приватное S3-совместимое объектное хранилище и восстанавливаются после обновления страницы, нового деплоя и на другом компьютере.

## Что изменилось

- Добавлен backend `src/order_persistence.py` для Cloudflare R2, AWS S3, Backblaze B2 и других S3-compatible сервисов.
- Крупные Excel загружаются через multipart transfer.
- Черновик сохраняется одновременно в локальный SQLite-кэш и в облако.
- При восстановлении облако имеет приоритет, локальный SQLite используется как резервный кэш.
- Список «Продолжить сохранённый заказ» объединяет облачные и сохранившиеся локальные записи.
- Старые локальные заказы 1.8.2–1.8.7 автоматически мигрируют в облако, если ещё существуют.
- Новый отчёт получает новый hash и чистое активное состояние, но предыдущий заказ больше физически не удаляется.
- JSON автосохранения стал разреженным: нулевые количества не отправляются в облако, поэтому сохранение остаётся быстрым даже на нескольких тысячах позиций.
- Добавлен полный переносимый ZIP: исходный Excel + черновик + manifest.
- В интерфейсе виден явный статус безопасного хранения.
- При `required = true` новый заказ нельзя начать, пока облачное хранилище недоступно.

## Совместимость

Формат входного отчёта и бизнес-логика версии 1.8.7 не менялись. Поддерживаются произвольные имена Excel, фактическая колонка NTR2, ТВП, именованные комплекты, точные названия камней и разделение заказов «Камни» / «Жемчуг».
