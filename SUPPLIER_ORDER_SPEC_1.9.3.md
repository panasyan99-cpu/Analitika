# Спецификация «Заказ поставщику» 1.9.3

## 1. Единая точка 20 / Princess Hang

Следующие заголовки означают один магазин:

- `20`;
- `20NDC`;
- `Princess Hang`;
- `Princess Hang Store`.

Внутреннее каноническое значение хранится в `stock_20`. Отдельного остатка Princess Hang в расчёте нет.

## 2. Общий остаток в карточке

```text
Общий остаток = Всего − stock_20
```

Правила:

- показывается всегда, включая ноль;
- 63 входит в общий остаток;
- TT/Outlet входит в общий остаток;
- магазин 20/Princess Hang не показывается отдельным кубиком;
- TT и 63 показываются как детализация и не вычитаются повторно.

## 3. Остаток для рекомендаций

```text
working_stock = Всего − stock_20 − stock_63
```

`working_stock` не выводится под названием «Общий остаток».

## 4. Несколько колонок-синонимов

Если в одном отчёте присутствует несколько колонок единой точки:

1. значения не суммируются;
2. выбирается максимальное неотрицательное значение;
3. при разных ненулевых значениях добавляется предупреждение;
4. баланс колонок и расчёт отсутствующего NTR2 используют дедуплицированное значение.

## 5. Магазин 63

Синонимы 63 остаются отдельной логикой:

- `63`;
- `63NDC`;
- `63.1`;
- `63.2`.

Разные физические колонки 63 могут суммироваться согласно существующей логике.

## 6. Контрольный пример

Для `KE15E518B-BS`:

```text
Всего = 4
20NDC = 0
TT = 1
63NDC = 1
Общий остаток = 4
working_stock = 3
```
