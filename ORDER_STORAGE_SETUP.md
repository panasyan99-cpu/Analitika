# Надёжное хранение заказов — настройка один раз

Версия 1.8.9 сохраняет исходный Excel, выбранные количества, ручной режим, Limited Order, размеры колец, отметки сверки и текущий этап в приватное S3-совместимое объектное хранилище.

Подойдут Cloudflare R2, AWS S3, Backblaze B2 и другие сервисы с S3 API. Локальный SQLite используется только как быстрый кэш и не считается единственным постоянным хранилищем.

## Важно: существующие Streamlit Secrets не удалять

Поле Secrets содержит **единый TOML-документ для всего приложения**. Если в нём уже есть Baserow, Supabase, курсы валют или другие ключи, оставьте их без изменений.

Новый блок добавляется **в самый низ**, после пустой строки:

```toml
# Все существующие секреты выше остаются как есть.

[order_storage]
endpoint_url = "https://<ACCOUNT_ID>.r2.cloudflarestorage.com"
access_key_id = "<ACCESS_KEY_ID>"
secret_access_key = "<SECRET_ACCESS_KEY>"
bucket = "analitika-orders"
region = "auto"
prefix = "princess/analitika/supplier-orders"
required = true
```

Не создавайте второй блок `[order_storage]`, если он уже есть: отредактируйте существующий.

## Вариант Cloudflare R2

1. Откройте Cloudflare Dashboard → **R2 Object Storage**.
2. Создайте приватный bucket `analitika-orders`.
3. В R2 откройте управление API Tokens.
4. Создайте токен с правами **Object Read & Write**, ограниченный этим bucket.
5. Скопируйте `Access Key ID`, `Secret Access Key` и S3 endpoint.
6. Добавьте блок `[order_storage]` в Streamlit Secrets и нажмите **Save**.

Секретный ключ обычно показывается один раз — сохраните его сразу.

## Где добавить в Streamlit Cloud

1. Откройте приложение.
2. Нажмите **Manage app** в правом нижнем углу.
3. Откройте меню `⋮` → **Settings** → **Secrets**.
4. Не стирая существующий текст, добавьте блок `[order_storage]` внизу.
5. Нажмите **Save** и дождитесь перезапуска приложения.

После запуска в разделе «Заказ поставщику» должна появиться зелёная строка о надёжном облачном сохранении.

## Что сохраняется

- исходный Excel с любым именем;
- отдельные заказы «Камни» и «Жемчуг»;
- подтверждённые и ручные количества;
- состояние «Изменить количество»;
- позиции Limited Order;
- размеры колец 15–24;
- отметки «С остатком сверился»;
- выбранный камень и текущий этап;
- предыдущие отчёты как восстанавливаемая история.

Сохранение происходит после каждого действия и по ручной кнопке «Сохранить черновик».

## Проверка перед работой

Не начинайте крупный заказ, пока не увидите статус:

> Надёжное сохранение включено

При `required = true` приложение блокирует загрузку нового заказа, если облачное хранилище недоступно.

## Аварийная копия

В активном заказе откройте «Черновик и резервная копия» и нажмите:

```text
Подготовить полный резерв: Excel + выбранные позиции
```

ZIP содержит исходный Excel, JSON черновика и manifest. Это дополнительная независимая копия.

## Локальный запуск

Для локального запуска создайте `.streamlit/secrets.toml` с тем же блоком. Этот файл нельзя коммитить в Git; он исключён через `.gitignore`.

Также поддерживаются переменные окружения:

```text
ORDER_STORAGE_ENDPOINT_URL
ORDER_STORAGE_ACCESS_KEY_ID
ORDER_STORAGE_SECRET_ACCESS_KEY
ORDER_STORAGE_BUCKET
ORDER_STORAGE_REGION
ORDER_STORAGE_PREFIX
ORDER_STORAGE_REQUIRED=true
```
