# Надёжное хранение заказов — настройка один раз

Версия 1.8.8 больше не считает диск Streamlit постоянным хранилищем. Исходный Excel, выбранные количества, размеры колец, отметки сверки и текущий этап заказа сохраняются в приватное S3-совместимое объектное хранилище.

Подойдут Cloudflare R2, AWS S3, Backblaze B2 или другой сервис с S3 API. Для крупных входных Excel рекомендуется хранилище, которое разрешает файлы больше 100 МБ.

## 1. Создайте приватный bucket

Рекомендуемое имя:

```text
analitika-orders
```

Публичный доступ не нужен.

## 2. Создайте ключ доступа

Ключ должен иметь права:

- читать объекты;
- записывать объекты;
- получать список объектов bucket.

Удаление приложению не требуется: старые заказы сохраняются как восстанавливаемая история.

## 3. Добавьте Streamlit Secrets

Откройте:

```text
Manage app → Settings → Secrets
```

Вставьте и заполните:

```toml
[order_storage]
endpoint_url = "https://<ACCOUNT_ID>.r2.cloudflarestorage.com"
access_key_id = "<ACCESS_KEY_ID>"
secret_access_key = "<SECRET_ACCESS_KEY>"
bucket = "analitika-orders"
region = "auto"
prefix = "princess/analitika/supplier-orders"
required = true
```

Для AWS S3 укажите регион bucket. `endpoint_url` можно удалить либо оставить пустым.

`required = true` блокирует начало нового заказа, если облачная копия не была создана. Это защищает от работы в небезопасном режиме.

## 4. Перезапустите приложение

В разделе «Заказ поставщику» должна появиться зелёная строка:

> Надёжное сохранение включено: Excel, выбранные позиции и размеры хранятся в облаке.

Только после этого начинайте новый заказ.

## Что сохраняется

- исходный Excel с любым именем;
- тип заказа: камни или жемчуг;
- все количества «К заказу»;
- распределение размеров колец 15–24;
- отметки «С остатком сверился»;
- выбранный камень и текущий этап;
- дата последнего изменения;
- предыдущие заказы как история.

Сохранение происходит после каждого изменения и по ручной кнопке. После обновления страницы, redeploy или открытия приложения на другом компьютере заказ появляется в блоке «Продолжить сохранённый заказ».

## Миграция старого локального черновика

Если файлы версии 1.8.2–1.8.7 ещё физически сохранились в `data/order_runtime`, версия 1.8.8 автоматически перенесёт их в облако при первом открытии списка сохранённых заказов.

Если предыдущий деплой уже удалил локальный диск, восстановить отсутствующие данные технически невозможно. Новая схема предотвращает такую потерю для последующих заказов.

## Аварийный переносимый резерв

Внутри активного заказа откройте «Черновик и резервная копия» и нажмите:

```text
Подготовить полный резерв: Excel + выбранные позиции
```

ZIP содержит исходный Excel, JSON черновика и контрольный manifest. Это дополнительная независимая копия; основной способ восстановления — облачное хранилище.

## Локальный запуск через переменные окружения

Вместо Streamlit Secrets можно использовать:

```text
ORDER_STORAGE_ENDPOINT_URL
ORDER_STORAGE_ACCESS_KEY_ID
ORDER_STORAGE_SECRET_ACCESS_KEY
ORDER_STORAGE_BUCKET
ORDER_STORAGE_REGION
ORDER_STORAGE_PREFIX
ORDER_STORAGE_REQUIRED=true
```

Секретные ключи нельзя добавлять в Git. Файл `.streamlit/secrets.toml` уже исключён через `.gitignore`.
