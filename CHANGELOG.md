## 1.4.0 — Sonu network stock

- Новый расширенный файл Sonu с полем «Остаток» принят как основной формат входного отчета.
- Остаток трактуется как общий по всей сети и может повторяться в строках одного SKU по разным магазинам; при агрегации он берется один раз, а продажи суммируются по всей сети.
- Добавлен контроль расхождений: если повторяющиеся значения остатка одного SKU не совпадают, система показывает предупреждение и формирует отдельный лист проверки.
- Для всех изделий, кроме браслетов, добавлена сетевая сводка «группа камня → вид камня → категория» с количеством SKU, общим остатком, продажами за период, выручкой и потребностью на 30/45/90 дней.
- Потребность рассчитывается компактно внутри фактических таблиц и карточек; отдельные перегруженные блоки прогноза, матрицы и приоритетов не возвращены.
- Браслеты разделены на «С центральной композицией» и «Полный круг». Первая группа включает модели с затяжкой и модели на кольцах, где камни сосредоточены преимущественно в середине и конструкция легче.
- Для новых или неоднозначных браслетов сохранено правило распределения 50/50; при нечетном количестве лишняя, более продаваемая модель относится к центральной композиции.
- В Sonu добавлен единый переключатель «Карточки / Таблица», который применяется к сетевой аналитике, магазинам, браслетам и списку SKU.
- Полная Excel-выгрузка пересобрана: сводка, сеть по группам, камни и категории, SKU сети, типы и модели браслетов, продажи по магазинам, контроль остатков и исходные данные.
- Шапка и раздел «О программе» обновлены под новую логику сетевых остатков и потребности.

## 1.1.15 — Concurrent comparison stability

- Parsed StoreData and pandas objects are cached per browser session instead of globally across users.
- Comparison files are submitted together in one form, preventing overlapping reruns while each file is selected.
- A process-wide thread-safe lock serializes the memory-intensive Excel parsing stage across simultaneous users.
- Streamlit fast reruns are disabled to avoid interrupting Excel parsing and Arrow/DataFrame rendering.
- Duplicate comparison launches are guarded and the comparison upload UI is removed after generation.
- Existing URL, business logic, tables, charts and comparison calculations are unchanged.

## 1.1.7 — Stability and memory optimization

- Тяжелые блоки магазинов, интерактивной аналитики и поставщиков переведены на `st.fragment`: изменение фильтра больше не перерисовывает весь отчет.
- Разобранный отчет хранится в общем ограниченном кэше с TTL 30 минут и максимум двумя версиями.
- Убраны отдельные копии разобранных данных из `session_state`.
- Выгрузка с поставщиками читается один раз вместо нескольких повторных открытий Excel.
- Скрытые вкладки заменены на ленивую детализацию: создается только выбранная таблица и выбранные диаграммы.
- После работы тяжелых фрагментов выполняется сборка временных объектов.
- Для production обязательно используется Python 3.12.
