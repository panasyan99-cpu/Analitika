# Changelog

## 1.2.3 — Единый курс сайта

- В обычный отчет и сравнение периодов добавлено поле «Курс VND за 1 USD» в боковой панели.
- Курс по умолчанию — 26 300 VND/USD; изменение курса сразу перестраивает денежные показатели.
- KPI, таблицы, горизонтальные и круговые диаграммы, абсолютные изменения и текстовые выводы отображаются в USD.
- Количество, доли и проценты не изменены.
- Обычный отчет, сравнение периодов, складская аналитика и «Заказ Sonu» используют один общий курс из session state.
- Архив поставляется как полный чистый репозиторий без установщиков и патчей.

## 1.2.2 — Sonu order analytics

- Добавлен отдельный раздел «Заказ Sonu» рядом с аналитикой сувениров.
- Поддержан расширенный отчет 1С: магазин, товар, камень/вставка, проба, номенклатурная группа и поставщик.
- Выведены продажи за период по магазинам, категориям, камням, пробам и моделям.
- Все денежные показатели пересчитываются в USD по редактируемому курсу; значение по умолчанию — 26 300 VND/USD.
- Возвраты полностью исключены. Скорость продаж означает количество изделий, проданных за отчетный период.
- Браслеты разделены на «в круг с камнями» и «с затяжкой» по фотографиям. Неоднозначные и новые модели распределяются 50/50; более продаваемая половина относится к «с затяжкой».
- Поскольку в исходной форме нет независимой колонки текущего остатка, показывается только расчетный остаток «отгружено − продано» с явным пояснением.


## 1.1.15 — Concurrent comparison stability

- Parsed StoreData and pandas objects are cached per browser session instead of globally across users.
- Comparison files are submitted together in one form, preventing overlapping reruns while each file is selected.
- A process-wide thread-safe lock serializes the memory-intensive Excel parsing stage across simultaneous users.
- Streamlit fast reruns are disabled to avoid interrupting Excel parsing and Arrow/DataFrame rendering.
- Duplicate comparison launches are guarded and the comparison upload UI is removed after generation.
- Existing URL, business logic, tables, charts and comparison calculations are unchanged.

## 1.1.14 — Compact release history

- В первый информационный блок добавлено правило для сравнительного режима: нужно выгрузить два одинаково настроенных отчета с одинаковыми уровнями группировки, изменив только период.
- История обновлений помещена в компактный блок с вертикальной прокруткой и больше не растягивает страницу «О платформе».
- Прокрутка поддерживает колесо мыши, трекпад и вертикальный жест на iPad.

## 1.1.13 — Comparison navigation state fix

- Боковая и мобильная навигация сравнительного режима теперь появляются сразу после первого запуска анализа.
- После установки флага готовности выполняется контролируемый rerun, поэтому UI и session state синхронизируются в одном цикле.
- Повторное нажатие кнопки «Запустить сравнительный анализ» больше не требуется.

## 1.1.12 — Separate GIFT TT direction

- GIFT TT больше не попадает в список сегментов сравнительного анализа.
- GIFT TT исключён из фильтров по сегменту, камню и номенклатурной группе.
- В сравнении выбранного магазина GIFT TT выводится только внутри OUTLET отдельной строкой GIFT TT ↔ GIFT TT.
- Основные ювелирные сегменты остаются неизменными: Top Stones, Pearls и Colored Stones.
- Обычный однофайловый отчёт OUTLET по-прежнему показывает GIFT TT отдельным блоком.

## 1.1.11 — Comparison workspace

- На стартовой странице добавлен отдельный режим **«Сравнение периодов»**.
- Обычный отчет работает как раньше и формируется автоматически сразу после загрузки файла.
- В сравнительном режиме используются два отдельных поля загрузки; обработка не начинается, пока не загружены оба файла и не нажата кнопка запуска.
- Система проверяет периоды, автоматически ставит более ранний период слева и не запускает сравнение одинаковых периодов.
- Добавлены сравнение сети, магазинов, сегментов, интерактивных срезов и поставщиков.
- Все сравнительные таблицы сохраняют числовые типы и корректную сортировку по выбранной колонке.
- Навигация в сравнительном режиме показывает только относящиеся к нему разделы.

## 1.1.7 — Stability and memory optimization

- Тяжелые блоки магазинов, интерактивной аналитики и поставщиков переведены на `st.fragment`: изменение фильтра больше не перерисовывает весь отчет.
- Разобранный отчет хранится в общем ограниченном кэше с TTL 30 минут и максимум двумя версиями.
- Убраны отдельные копии разобранных данных из `session_state`.
- Выгрузка с поставщиками читается один раз вместо нескольких повторных открытий Excel.
- Скрытые вкладки заменены на ленивую детализацию: создается только выбранная таблица и выбранные диаграммы.
- После работы тяжелых фрагментов выполняется сборка временных объектов.
- Для production обязательно используется Python 3.12.
