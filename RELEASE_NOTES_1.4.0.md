# Analitika Web 1.4.0 — Sonu network stock

## Что изменилось

- Поддержан новый файл Sonu с общим остатком сети в колонке `Остаток`.
- Остаток повторяется по магазинам, но учитывается один раз на SKU.
- Продажи суммируются по всей сети.
- Для не-браслетов добавлена сводка по виду камня и категории изделия.
- Рассчитывается потребность на 30, 45 и 90 дней с учетом фактического остатка.
- Браслеты разделены на центральную композицию и полный круг.
- Добавлен общий переключатель карточек и таблиц.
- Excel-выгрузка полностью пересобрана под сетевой остаток.

## Формула потребности

Для каждого SKU:

`max(ceil(продажи за период / дни периода × горизонт) − остаток сети, 0)`

Расчет выполняется на уровне SKU, после чего значения суммируются по камню, категории или типу браслета.

## Контроль остатка

Если один SKU содержит разные значения общего остатка в строках магазинов, система не складывает их, показывает предупреждение и добавляет SKU в лист `Контроль остатков`.
